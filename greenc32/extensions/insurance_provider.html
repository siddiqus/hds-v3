hData Extension Description: Insurance Provider

This documentation describes the hData content payload for the Insurance Provider Extension.

1. Identification

Name Insurance Provider Extension Description
Version Aware URL
Creation Date 20 July 2012
Summary This Insurance Provider hData Extension Description contains information about the entities or other individuals who may pay for a patient's healthcare. Such entities or individuals may be health insurance plans, other payers, and guarantors, parties with financial responsibility, some combination of payers or the patient directly. This module is used to define which entity or combination of entities has any financial responsibility for a patient's care.

3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Insurance Provider contains information about the entities or individuals with financial responsibility for a patients care.

4.3. Clinical Requirement

Insurance Provider is characterized by:

  • the payer
  • the subscriber
  • the guarantor
