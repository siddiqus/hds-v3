3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Social History contains information about the current and relevant historical social history observations for the patient..

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Entry Component: An identifier for this specific observation
code code O Entry Component: This is a coded value indicating the type of social history observation The Social History type SHALL be coded as specified in HITSP/C80 Section 2.2.2.4 Social History Type
free text string M Entry Component: This is a text description of the social history.
value quantity O

A value describing the social history

4.3. Clinical Requirement

Social History is characterized by:

  • entry of the social history observation
  • value of the social history observation

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Social History specifies how to formally describe the elements in an XML document. For the RHEx Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated.

Social_History.xsd is located at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/social_history.xsd

Social_History.xsd extends the entry complex data type which is defined in the data_types schema at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/data_types.xsd

4.5. Resource Representations

Social History may be represented in XML and JSON format.
