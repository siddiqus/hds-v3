3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Advance Directive contains information about the existence of living wills, healthcare proxies and CPR and resuscitation status for a patient.

4.2. Description

NameTypeMandatory/OptionalNotes
code code M

This is a coded value describing the type of the Advance Directive
freeText string M Free text comment to describe the Advance Directive Type
Effective Time interval M The effective date for the Advance Directive

4.3. Clinical Requirement

Advance Directive is characterized by:

  • coded value of the advance directive
  • effective date of the advance directive
  • description of the advance directive

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Allergy specifies how to formally describe the elements in an XML document. For the RHEx Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated.

Advance_Directive.xsd is located at the following URI: https://github.com/projectcypress/health-data-standards/tree/develop/greenc32/schemas/rest_plus/advance_directive.xsd

Advance_Directive.xsd extends the entry complex data type which is defined in the data_types schema at the following URI: https://github.com/projectcypress/health-data-standards/tree/develop/greenc32/schemas/rest_plus/data_types.xsd

4.5. Resource Representations

Advance Directive may be represented in XML and JSON format.
