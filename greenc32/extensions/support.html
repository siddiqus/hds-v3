3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Support contains information about the patient's sources of support, such as immediate family, relatives and guardians

4.3. Clinical Requirement

Support is characterized by:

  • entry of the support observation
  • contact information of the support observation

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Support specifies how to formally describe the elements in an XML document. For the RHEx Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated.

Support.xsd is located at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/support.xsd

Support.xsd extends the entry complex data type which is defined in the data_types schema at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/data_types.xsd

4.5. Resource Representations

Support may be represented in XML and JSON format.
