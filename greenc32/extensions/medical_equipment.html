3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Medical Equipment contains information about the existence of living wills, healthcare proxies and CPR and resuscitation status for a patient.

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Entry component: Unique id for the medical equipment entry
code code O

Entry component: The coded value of the Medical Equipment entry represented
status string O

Entry component: The status of the Medical Equipment entry.

name string M Entry Component: FreeText - This is the name or other description of the Medical Equipment.
eventDate DateTime O Entry Component: effectiveDate - This is a date that expresses when this particular Medical Equipment entry was known to be active for the patient.
quantity quantity O

Numeric quantity of the Medical Equipment item associated
manufacturer organization O The organizational entity that is associated with manufacturing the medical equipment item

4.3. Clinical Requirement

Medical Equipment is characterized by:

  • entry of the medical equipment item
  • quantity of the medical equipment item
  • manufacturer of the medical equipment item

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Medical Equipment specifies how to formally describe the elements in an XML document. For the RHEx Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated.

Medical_Equipment.xsd is located at the following URI: https://github.com/projectcypress/health-data-standards/tree/develop/greenc32/schemas/rest_plus/medical_equipment.xsd

Medical_Equipment.xsd extends the entry complex data type which is defined in the data_types schema at the following URI: https://github.com/projectcypress/health-data-standards/tree/develop/greenc32/schemas/rest_plus/medical_equipment.xsd

4.5. Resource Representations

Medical Equipment may be represented in XML and JSON format.
