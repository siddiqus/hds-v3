3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Vital Sign contains information about the current and relevant historical vital sign observations for the patient..

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Entry Component: An identifier for this specific observation
code code O

Entry Component: The coded value of the vital sign represented. SHOULD be selected from LOINC (codeSystem 2.16.840.1.113883.6.1): 9279-1 Respiratory Rate, 8867-4 Heart Rate, 2710-2 O2 % BldC Oximetry, 8480-6 BP Systolic, 8462-4 BP Diastolic, 8310-5 Body Temperature, 8302-2 Height, 8306-3 Height (Lying), 8287-5 Head Circumference, 3141-9 Weight Measured
status string M

Entry Component: Status for this observation, e.g., complete, preliminary

name string M Entry Component: This is a free text description of the vital sign.
effectiveTime TimeInterval M Entry Component: The biologically relevant date/time for the observation.
value quantity O

The value of the vital sign, including units of measure if applicable
interpretation simple code O An abbreviated interpretation of the observation, e.g., normal, abnormal, high, etc.
reference range any O Reference range(s) for the observation

4.3. Clinical Requirement

Vital Sign is characterized by:

  • entry of the observation
  • value of the vital sign observation
  • interpretation of the vital sign observation
  • reference range of the vital sign observation

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Vital Sign specifies how to formally describe the elements in an XML document. For the RHEx Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated.

Vital_Sign.xsd is located at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/vital_sign.xsd

Vital_Sign.xsd extends the entry complex data type which is defined in the data_types schema at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/data_types.xsd

4.5. Resource Representations

Vital Sign may be represented in XML and JSON format.
