3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Result contains information about the current and relevant historical result observations for the patient..

4.2. Description

The value of the result, including units of measure if applicable
interpretation simpleCode O An abbreviated interpretation of the observation, e.g., normal, abnormal, high, etc. Result Interpretation SHALL be coded as specified in HITSP/C80 Section 2.2.3.6.3 Result Normalcy Status
reference range any O Reference range(s) for the observation

4.3. Clinical Requirement

Result is characterized by:

  • entry of the observation
  • value of the result observation
  • interpretation of the result observation
  • reference range of the result observation

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Result specifies how to formally describe the elements in an XML document. For the RHEx Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated.

Result.xsd is located at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/result.xsd

Result.xsd extends the entry complex data type which is defined in the data_types schema at the following URI: https://github.com/projectcypress/health-data-standards/blob/develop/greenc32/schemas/rest_plus/data_types.xsd

4.5. Resource Representations

Result may be represented in XML and JSON format.
